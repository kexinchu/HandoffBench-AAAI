# HandoffBench related-work and novelty audit

**Search cutoff:** 2026-07-18. **Evidence policy:** entries below use original papers, official project repositories, or official framework documentation. arXiv DOI links are included where available. Absence claims are deliberately scoped to the sources found by this search and should not be phrased as universal priority claims.

## Executive novelty verdict

The broad idea “benchmark what happens when one agent hands work to another” is **not novel**. The closest collision is *Handoff Debt* (June 2026), which interrupts coding agents at deterministic points and evaluates successor agents under repository-only, raw-trace, summary-note, and structured-note views. It directly occupies the words *handoff benchmark*, *successor agent*, and *structured notes*.

The defensible HandoffBench position is narrower and still meaningful:

> a controlled benchmark of **semantic state fidelity at inter-role control-transfer boundaries** in conversational, tool-using workflows, with a gold state specification at the boundary and safety/transaction-sensitive state categories (verified evidence, unresolved slots, consent, commitments, risk flags, and action preconditions), plus causal measurement of downstream regression.

This differs from coding-task resumption (*Handoff Debt*), end-state tool-agent benchmarks (tau-bench, ToolSandbox, STATE-Bench), general multi-agent collaboration benchmarks (MultiAgentBench), and cross-task procedural memory (LEGOMem). The paper should not claim the first handoff benchmark or the first structured handoff artifact. A safer claim is: **“To our knowledge, the first benchmark that isolates and scores field-level transfer of task-critical state at LLM agent handoff boundaries in interactive, policy-constrained workflows.”** Even that claim must be re-checked immediately before submission.

## Novelty collisions and how to survive them

1. **“First handoff benchmark” — untenable.** *Handoff Debt* already uses that exact framing. State a scoped first claim or omit priority language.
2. **“Structured handoff beats summary” — weak/occupied.** *Handoff Debt* already compares structured notes and summary notes; OpenAI supports typed handoff inputs; LangGraph supports typed persistent state. The contribution must be **gold semantic state + provenance/check semantics + causal downstream analysis**, not JSON formatting.
3. **“Stateful enterprise benchmark” — occupied.** tau-bench, ToolSandbox, and STATE-Bench cover this. HandoffBench's experimental variable must be the **transport across a fixed boundary**, with upstream trace and downstream agent held constant.
4. **“Multi-agent coordination benchmark” — crowded.** MultiAgentBench and successors own broad collaboration/topology evaluation. Avoid topology sweeps as the headline.
5. **“Verified/shared state” — partially occupied.** DeLM uses compact verified shared updates. Defend capsules through boundary-specific authority, provenance to user/tool/policy events, explicit unknown/contradicted status, and executable precondition blocking.
6. **“Memory for multi-agent workflows” — occupied and contrary to brief.** LEGOMem and STATE-Bench make this especially risky. Use only same-episode state; reset all cross-task memory; call shared memory a baseline, not the thesis.
7. **Domain collision.** Travel/refund/shopping are heavily used by tau-bench and STATE-Bench. Novelty cannot rest on domains. Either build new scenarios or explicitly derive a handoff intervention layer over established task semantics and emphasize boundary annotations.
8. **“Provenance-aware safety” --- occupied.** ProvenanceGuard already formalizes whether a tool call is supported by traceable evidence and uses that signal for blocking. HandoffBench must treat provenance as an experimentally manipulated boundary representation factor, report advisory and enforced conditions separately, and avoid claiming provenance analysis itself as novel.
9. **“State probes reveal hidden failure” --- occupied.** NeuroState-Bench already uses side-query probes for commitment integrity. The distinction must be a controlled predecessor--successor boundary, authenticated field provenance, and transport interventions rather than the generic value of probes.

## Recommended claim and experiment language

### Safe contribution statement

“We introduce a controlled evaluation protocol that inserts an agent-ownership boundary into interactive tool workflows and annotates the task-critical state that must cross it. Unlike end-state agent benchmarks and prior takeover evaluation, the protocol scores field-level state recall, precision, contradiction, unsupported commitments, and missed action preconditions, then links those errors to downstream behavior.”

### Causal design reviewers will expect

- Freeze the identical upstream trace at the handoff boundary and replay it across every transport condition.
- Use the identical target model, prompt, tools, simulator seed, and decoding settings across conditions.
- Include an **oracle gold-state handoff** upper bound. Without it, capsule generation error and target-agent use error are confounded.
- Include a **no-handoff continuous single-agent** reference, but do not interpret it as a perfectly matched architecture comparison unless capabilities/context are controlled.
- Factorial ablation: typing only; + provenance pointers; + executable precondition checks. This is essential because official frameworks already offer typed payloads.
- Separate three stages: source-state extraction fidelity, transport fidelity, and target utilization fidelity. A field being present does not mean the receiver uses it.
- Deterministically score facts, consent, tool calls, and final database state where possible; reserve LLM judges for open-ended UX only.
- Adversarial boundary tests should include conflicting later evidence, unresolved-but-plausible slots, scope-limited consent, stale tool results, negative facts, and an action whose precondition is absent.
- Report paired confidence intervals/effect sizes, per-domain and per-state-category results, and accuracy-cost Pareto curves; BenchAgent makes protocol alignment a likely reviewer concern.

## Suggested Related Work narrative

Agent evaluation first focused on single agents acting in diverse interactive environments (AgentBench), while tau-bench and ToolSandbox introduced dynamic user interaction, policy constraints, stateful tools, and executable end-state or milestone checks. STATE-Bench extends realistic enterprise environments toward cross-task learning and memory. These benchmarks establish that state and rules matter, but they do not intervene on how a fixed within-task state is transported when task ownership changes.

MultiAgentBench and BattleAgentBench evaluate collaboration, competition, and communication protocols at team level. LEGOMem studies where reusable procedural memory should be placed in a multi-agent workflow, and DeLM studies compact verified updates in shared context. HandoffBench instead isolates a single semantic interface between a predecessor and successor and evaluates whether task-critical facts, unknowns, permissions, and preconditions survive that interface.

The closest work is *Handoff Debt*, which measures the rediscovery cost of coding-agent takeover under repository, trace, summary, and structured-note views. HandoffBench should credit it as establishing takeover-view evaluation, then distinguish its own focus: role-specialized conversational/tool workflows; a field-level gold boundary state; evidence, consent, commitment, and precondition errors; and causal linkage between transfer errors and unsafe or regressive downstream behavior. Official OpenAI, LangChain, Microsoft, and AutoGen implementations further show that real frameworks expose materially different context-transfer semantics, motivating evaluation without itself constituting a novel transfer method.

## Bottom line for an AAAI reviewer

The project is viable only if the benchmark is diagnostic rather than another set of agent tasks. Its strongest scientific object is the **handoff boundary as an independently manipulated interface**. The executable capsule is credible as a method only if provenance/checks yield gains beyond a schema-matched typed payload and if the benchmark can localize why those gains occur. If experiments merely show that well-written JSON summaries outperform prose, novelty is below the target bar after *Handoff Debt* and current framework support.
